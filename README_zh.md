Fibersync

**为 Fabric 打造的高性能“热回档”备份解决方案**

Fibersync 是一款专为 Fabric 生态设计的备份与回档工具。与其它的回档插件相比，Fibersync 最大的特色在于其核心的**“热回档”**技术，旨在提供无缝、直观且安全的服务器管理体验。

--------------------------------------------------------------------------------

✨ 核心特性

- **🔥 真正的“热回档”** 回档操作**无需重启服务器**，在线玩家**无需断开连接**。这一特性完美解决了回档后需要重新挂载假人（Carpet Bots）的烦恼。
- **📊 可视化进度管理** 所有备份与回档操作均配备**进度条展示**，让复杂的操作过程清晰可见。
- **💾 多备份槽位与锁定机制** 支持创建多个备份槽位，并提供**锁定功能**防止重要备份被意外删除。
- **🌍 全场景支持** 无论是**多端服务器**还是**单机存档**，Fibersync 都能稳定运行。
- **🛡️ 安全确认流程** 执行回档或删除等危险操作时，系统会生成**随机确认码**，确保每一次操作都经过深思熟虑。

--------------------------------------------------------------------------------

🚀 快速开始

安装

1. 确保您的环境已安装 **Fabric Loader**。
2. 将下载的 `.jar` 文件放入服务器或客户端的 `mods` 文件夹内。
3. 启动游戏或服务器，程序将自动生成 `fibersync.json` 配置文件。

基础指令

| 指令                | 说明                          |
| ----------------- | --------------------------- |
| `/fs make [描述]`   | 创建一个系统自动命名的备份（如 b1, b2...）。 |
| `/fs back [备份名]`  | 回档到指定备份。若不填则默认回档至最新备份。      |
| `/fs list`        | 查看备份列表，包含时间、描述和文件大小。        |
| `/fs lock <备份名>`  | 锁定备份，防止被系统或手动删除。            |
| `/fs confirm <码>` | 输入系统提示的确认码以执行回档或删除。         |

--------------------------------------------------------------------------------

🛠️ 进阶功能

自定义备份

如果您需要为备份赋予特定的名称，可以使用： `/fs create <备份名> <描述>`。_建议：备份名请使用字母、数字或下划线，避免使用空格。_

管理员模式 (Showhand)

如果您在进行调试，觉得二次确认过于繁琐，可以开启“无需确认”模式：

- **开启：** `/fs showhand` — 开启后所有操作将直接执行，不再提示确认码。
- **关闭：** `/fs showhand stop` — 恢复安全确认模式。

镜像模式 (Mirror Mode)

对于高级用户，Fibersync 支持通过配置 `syncDir` 来实现镜像同步：

- 使用 `/fs sync <备份名>` 同步指定镜像，甚至支持按维度（Dimension）进行同步。
- 使用 `/fs list sync` 查看可用的镜像列表。

--------------------------------------------------------------------------------

⚙️ 配置文件

在 `fibersync.json` 中，您可以自定义指令前缀。例如，若将前缀修改为 `bf`，则所有 `/fs` 指令都将变更为 `/bf`。
